HTML

The foundational structure of a website. Made up of elements

Elements

Describes the 'type' of content on a webpage

Can also divide and describe the page to browsers and developers

Semantic Elements:

Elements where the name shows off what it's supposed to do

Child Elements:

the elements that are inside of 1 bigger element

The two most important elements and their children:

Head

Head is where the metadata goes; it tells the browser about the webpage

Head's Children:

  • meta: The metadata. metadata is the stuff that tells your browser how to display your website(descriptions, author, keywords, viewport)
  • title: the title of the webpage
  • style: the CSS styles associated with the webpage
  • base: the URL for the webpage
  • link: connects an external resource to the page
  • charset: tells the browser what characters are allowed on the page

Body

Body is where the content that the user sees will go - the 'body' of the page.

Body's Children:

  • h1, h2, h3, h4, h5, and h6: Headers of various length; h1 should only appear on a page once.
  • p: Paragraph
  • ul: unordred list(bullets)
  • ol: ordered list(numbers)
  • li: list item
  • b: bold
  • i: italic
  • u: underline
  • img: contains info about an image
  • br: break
  • a: anchor (adds a link)
  • section: semantic elements that helps define and separate content

Git

The interface that we use to code

Basic CLI Prompts:

  • pwd:tells you the folder you are currently in
  • ls:shows the files that are in the folder you are currently in
  • cd:'change directory', navigates through folders
  • mkdir:makes a new folder
  • touch:makes a new file inside the folder
  • start:opens a file

Git Workflow:

  • git checkout -b (branch-name) : Make a new Git Branch
  • code
  • git add -A : add something to the current branch
  • git commit -m "Text describing the changes you made" : Commit changes
  • git pull origin main : Sync local branch w/ base branch on Github
  • git push origin (branch-name) : pushing stuff to Github
  • click 'compare and pull request' on Github
  • make a request for colleagues to review
